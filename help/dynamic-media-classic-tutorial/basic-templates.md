# 基本模板簡介 {#basic-templates}

在Dynamic Media Classic語中，模板是在模板發佈後可通過URL動態更改的文檔。 Dynamic Media Classic提供基本模板，即從影像伺服器調用的基於影像的模板，由影像和渲染的文本組成。

模板最強大的一個方面是它們具有直接整合點，使您能夠將它們與資料庫綁定。 因此，您不僅可以提供影像並調整其大小，還可以查詢資料庫以查找新項目或銷售項目，並使這些項目在影像上顯示為覆蓋。 您可以要求提供項目的說明，並使其以您選擇的字型和佈局顯示為標籤。 可能性是無限的。

基本模板可以通過從簡單到複雜的多種不同方式實現。 例如：

- 基本商品。 如果產品有免費運輸，則使用「免費運輸」等標籤。 這些標籤是由Photoshop的商品團隊設定的，而Web使用邏輯來知道何時將它們應用於影像。
- 高級商品銷售。 每個模板具有多個變數，並可同時顯示多個選項。 使用資料庫、清單和業務規則確定何時將產品顯示為「Just In」、「Clearance」或「Sold Out」。 還可以使用產品背後的透明度來顯示不同背景，如在不同的房間。 相同的模板和/或資產可以在產品詳細資訊頁面上重新使用，以顯示不同背景下相同產品的較大或可縮放版本。

必須瞭解的是，Dynamic Media Classic只提供這些基於模板的應用程式的可視部分。 Dynamic Media Classic公司或其整合合作夥伴必須提供業務規則、資料庫和開發技能以建立應用程式。 沒有「內置」模板應用程式；設計人員在Dynamic Media Classic設定模板，開發人員使用URL調用來更改模板中的變數。

在本教程的本節結束時，您將瞭解如何：

- 將PhotoshopPSD上載到Dynamic Media Classic以將其用作模板的基礎。
- 建立由影像層組成的簡單促銷基本模板。
- 添加文本層，並通過使用參數使其變數。
- 構建模板URL並通過Web瀏覽器動態地處理影像。

>[!NOTE]
>
>本章中的所有URL僅用於說明目的；它們不是即時連結。

## 基本模板概覽

基本模板（簡稱「模板」）的定義是URL可定址的分層影像。 最終結果是影像，但可通過URL更改。 它可以包含照片、文本或圖形 — 任何PTIFF資產在Dynamic Media Classic的組合。

模板與PhotoshopPSD檔案最相似，因為它們具有相似的工作流和類似的功能。

- 兩者都由層組成，就像堆疊的醋酸鹽片。 可以合成部分透明的影像，並通過圖層的透明區域查看下面的圖層。
- 可以移動和旋轉圖層以重新定位內容，並且可以改變不透明度和混合模式以使內容部分透明。
- 可以建立基於文本的圖層。 因為影像伺服器使用的文本引擎與Photoshop和Illustrator相同，所以質量可能非常高。
- 簡單圖層樣式可應用於每個圖層，以建立特殊效果，如投影或灰度。

但是，與PhotoshopPSD不同，圖層可以完全動態，並通過影像伺服器上的URL進行控制。

- 可以向所有模板屬性添加變數，使其可以輕鬆地動態更改組合。
- 調用參數的變數允許您僅公開要更改的模板部分。

您只需為每個會有所不同的圖層添加一個佔位符，而不是像在Photoshop那樣將所有圖層放在單個檔案中，並顯示和隱藏它們（儘管您也可以這樣做，如果願意的話）。

使用佔位符，您可以動態地將圖層的內容與另一個已發佈的資產交換，並且它將自動獲得它所替換圖層的相同屬性（如大小和旋轉）。

由於基本模板通常在Photoshop設計，但是通過URL部署，因此模板項目需要兼顧設計和技術技能。 我們通常假設從事創意模板工作的人是Photoshop設計師，而實施模板的人是Web開發人員。 創意團隊和開發團隊必須密切合作，才能使模板取得成功。

模板項目可能相對簡單或極其複雜，具體取決於業務規則和應用程式的需要。 基本模板是從映像伺服器調用的，但是，由於Dynamic Media Classic環境的靈活性，您甚至可以將模板嵌套在其他模板中，從而可以建立相當複雜的影像，這些影像可以通過通常命名的變數進行連結。

- 瞭解有關 [模板基礎知識](https://experienceleague.adobe.com/docs/dynamic-media-classic/using/template-basics/quick-start-template-basics.html)。
- 瞭解如何建立 [基本模板](https://experienceleague.adobe.com/docs/dynamic-media-classic/using/template-basics/creating-template.html#creating_a_template)。

## 建立基本模板

使用「基本模板」時，通常遵循下圖中的工作流步驟。 如果使用動態文本圖層，則用虛線標籤的步驟是可選的，並在下面的說明中指示為「文本工作流」。 如果不使用文本，請僅遵循主路徑。

![影像](assets/basic-templates/basic-templates-1.png)

_基本模板工作流。_

1. 設計並建立資產。 大多數用戶在Adobe Photoshop都這樣做。 按您需要的確切大小設計資產 — 如果是縮略圖頁面的200像素影像，則以200像素設計資產。 如果需要放大它，請將其設計為大約2000像素。 使用Photoshop(和/或以點陣圖形式保存的Illustrator)建立資產，並使用Dynamic Media Classic將部件組合在一起、管理圖層和添加變數。
2. 設計完圖形資產後，將其上傳到Dynamic Media Classic。 我們建議您上載整個分層PSD檔案，並讓Dynamic Media Classic通過使用以下方式為每個層建立一個檔案，而不是從PSD上載單個資產 **維護層** 上載時的選項（有關詳細資訊，請參閱下面）。 _文本工作流：如果建立動態文本，也上載字型。 動態文本是可變的，並通過URL進行控制。 如果您的文本是靜態的，或只有幾個短語沒有更改 — 例如，標籤說「新建」或「銷售」，而不是「X%關閉」（X為變數數） — 我們建議在Photoshop預呈現文本，並將其作為柵格化圖層上傳為影像。 它更簡單，而且您可以按自己的要求完全樣式文本。_
3. 使用「生成」菜單的「模板基礎」編輯器在Dynamic Media Classic生成模板並添加影像層。 文本工作流：在同一編輯器中建立文本層。 在Dynamic Media Classic手動構建模板時需要此步驟。 選擇與設計相匹配的畫布大小，將影像拖放到畫布上，並設定圖層屬性（大小、旋轉、不透明度等）。 您不會將每個可能的圖層都放在模板上，而只是每個影像圖層一個佔位符。 _文本工作流：使用「文本」工具建立文本圖層，類似於在Photoshop建立文本圖層。 可以使用「Photoshop類型」工具中提供的相同選項來選擇字型和樣式。_ 另一個工作流是上載PSD，讓Dynamic Media Classic生成「免費」模板，甚至可以重新建立文本層。 稍後將詳細討論此問題。
4. 建立圖層後，將參數（變數）添加到要通過URL控制的任何圖層的任何屬性，包括圖層的源（影像本身）。 _文本工作流：您還可以向文本圖層添加參數，以控制文本內容以及圖層本身的大小和位置，以及所有格式設定選項，如字型顏色、字型大小、水準跟蹤等。_
5. 建立與模板大小匹配的影像預設。 我們建議這樣做，這樣模板總是以1:1大小調用，並且還要向任何大影像圖層添加銳化功能，這些影像圖層將調整大小以適應模板。 如果要生成要放大的模板，則不必執行此步驟。
6. 發佈、從Dynamic Media Classic預覽複製URL，並在瀏覽器中test它。

## 準備模板資產並將其上載到Dynamic Media Classic

在將模板資產上載到Dynamic Media Classic之前，您需要完成一些準備步驟。

### 準備PSD以上載

在將Photoshop檔案上傳到Dynamic Media Classic之前，請簡化Photoshop的各層，使其更易於使用，並與映像伺服器具有最大的相容性。 您的PSD檔案通常由Dynamic Media Classic無法識別的許多元素組成，而您最終可能還會有許多難以管理的小部分。 請確保保存主PSD的備份，以備以後需要編輯原始檔案。 您將上載簡化副本，而不是主副本。

![影像](assets/basic-templates/basic-templates-2.jpg)

1. 通過將需要開啟/關閉的相關層合併/平整為單個層來簡化層結構。 例如，標籤「NEW」和藍色橫幅將合併到單個圖層中，這樣您只需按一下即可顯示或隱藏它們。
   ![影像](assets/basic-templates/basic-templates-3.jpg)
2. Dynamic Media Classic或影像伺服器不支援某些圖層類型和圖層效果，因此在上載前需要柵格化。 否則，可能會忽略效果，或丟棄圖層。 柵格化圖層意味著將圖層從可編輯轉換為不可編輯。 要柵格化圖層效果或文本圖層，請建立空圖層，同時選擇兩者並使用 **層>合併層** 或CTRL + E/CMD + E。

   - Dynamic Media Classic不能對層進行分組或連結。 組或連結集中的所有圖層都轉換為不再分組/連結的單獨圖層。
   - 圖層蒙版在上載時會轉換為透明度。
   - 不支援調整層，並且將丟棄。
   - 填充層（如「實色」層）被柵格化。
   - 在上載時，智慧對象圖層和向量圖層柵格化為正常影像，並應用和柵格化智慧濾鏡。
   - 除非使用「提取文本」(Extract Text)選項，否則文本層也將被柵格化 — 有關詳細資訊，請參閱下面。
   - 大多數圖層效果都被忽略，只支援少數混合模式。 如果有疑問，請在Dynamic Media Classic添加簡單效果（如內部或投影、內部或外部光暈），或使用空白圖層合併和柵格化Photoshop中的效果。

### 使用字型

如果需要生成動態文本，還將上載和發佈字型。 Dynamic Media Classic中唯一包含的字型是Arial。

每個公司都有責任獲取在Web上使用字型的許可證 — 只要在您的電腦上安裝字型，您就無權在Web上進行商業使用，如果未經許可使用，您的公司可能會面臨來自字型發佈商的法律訴訟。 此外，許可條款也會有所不同 — 例如，您可能需要單獨的許可證才能進行打印和螢幕顯示。

Dynamic Media Classic支援標準OpenType(OTF)、TrueType(TTF)和Type 1 Postscript字型。 Mac — 只支援手提箱字型、文字收藏檔案、Windows系統字型和專有機器字型（如雕刻機或刺繡機使用的字型） — 您必須將它們轉換為標準字型格式之一或替換Dynamic Media Classic和影像伺服器上使用的類似字型。

在將字型上載到Dynamic Media Classic後，與任何其他資產一樣，它們還必須發佈到影像伺服器。 最常見的模板錯誤是忘記發佈字型，這將導致影像錯誤 — 影像伺服器不會替換其它字型。 此外，如果您想 **提取文本** 選項上載時，必須先上載字型檔案，然後才能上載使用這些字型的PSD。 的 **提取文本** 功能將嘗試將文本重新建立為可編輯文本圖層，並將其放在Dynamic Media Classic模板中。 下一個主題「PSD選項」將討論此問題。

請注意，字型具有多個內部名稱，這些名稱通常與其外部檔案名不同。 您可以在Dynamic Media Classic的該資產的「詳細資訊」頁面上查看其所有不同名稱。 以下是字型Adobe CaslonPro Semibold的名稱，列在Dynamic Media Classic的「元資料」頁籤下：

![影像](assets/basic-templates/basic-templates-4.jpg)

_Dynamic Media Classic中字型的「詳細資訊」頁面上的元資料頁籤。_

Dynamic Media Classic將此字型(ACaslonPro-Semibold)的檔案名用作其資產ID，但該名稱不是模板使用的名稱。 模板使用底部列出的RTF格式名稱。 RTF是Image Server文本引擎的本機「語言」。

如果需要通過URL更改字型，必須調用字型的RTF名稱（而不是資產ID），否則將出現錯誤。 在這種情況下，此字型的正確名稱是「Adobe Caslon專業」。 我們將在下面的「RTF和文本參數」主題中討論有關字型和RTF的詳細資訊。

在Windows和Mac系統上找到的最常見的字型檔案格式是OpenType和TrueType。 OpenType具有.OTF副檔名，而TrueType為.TTF。 兩種模式在Dynamic Media Classic同樣有效。

### 上載PSD時選擇選項

您不需要上載Photoshop檔案(PSD)來建立模板；模板可以由Dynamic Media Classic的任何影像資產構建。 但是，上載PSD可以使創作更加容易，因為通常，您會將這些資產放在分層PSD中。 此外，Dynamic Media Classic將在您上傳分層PSD時自動生成模板。

- **維護層。** 這是最重要的選擇。 這告訴Dynamic Media Classic為每個Photoshop層建立一個影像資源。 如果未選中，則禁用所有其它選項，並將PSD拼合為單個影像。
- **建立** **模板。** 此選項將生成的各層取回，並通過將它們組合在一起自動建立模板。 使用自動生成的模板的一個缺點是，Dynamic Media Classic將所有圖層放在一個檔案中，而我們只需要每個圖層一個佔位符。 刪除額外的圖層非常容易，但如果您有多個圖層，則重新建立這些圖層會更快。 請務必更名新模板；如果未上載，則下次重新上載同一PSD時，該操作將被覆蓋。
- **提取文本。** 這會使用您上載的字型將PSD中的文本圖層重新建立為模板中的文本圖層。 如果文本在Photoshop的路徑上並且要在模板中維護該路徑，則此步驟是必需的。 此功能要求您使用 **建立模板** 選項，因為提取的文本只能由上載時生成的模板建立。
- **將圖層擴展到背景大小。** 此設定使每個圖層的大小與整個PSD畫布的大小相同。 對於始終保持固定位置的層，此選項非常有用：否則，在將影像交換到同一圖層時，可能必須重新放置影像。
- **層命名。** 這將告訴Dynamic Media Classic如何命名每層生成的每個資產。 我們建議 **Photoshop** **和層** **名稱** 或者Photoshop **層** **數字**。 兩個選項都將PSD名稱用作名稱的第一部分，並在末尾添加層名稱或編號。 例如，如果您有一個名為「shirt.psd」的PSD，並且它有名為「front」、「sleeps」和「collar」的圖層，則使用 **Photoshop** 層 **名稱** 選項，Dynamic Media Classic將生成Asset ID &quot;shirt_front&quot; 、 shirt_lieves&quot;和shit_collar。&quot; 使用其中一個選項有助於確保名稱在Dynamic Media Classic是唯一的。

## 使用影像層建立模板

儘管Dynamic Media Classic可以從分層PSD自動建立模板，但您應該知道如何手動構建模板。 如上所述，有時您不想使用Dynamic Media Classic建立的模板。

### 模板基礎UI

我們先熟悉一下編輯介面。

在左中心是您的工作區，顯示最終模板的預覽。 右側是「圖層」和「圖層屬性」面板。 這些領域是你最擅長的工作。

![影像](assets/basic-templates/basic-templates-5.jpg)

_「生成模板基礎資訊」頁。_

- **預覽/工作區。** 這是主窗。 在此，您可以使用滑鼠移動、調整圖層大小和旋轉圖層。 層輪廓顯示為虛線。
- **層。** 這與「Photoshop層」面板類似。 將圖層添加到模板時，它們將出現在此。 圖層從上到下堆疊 — 「圖層」面板中的頂層位於清單中「圖層」面板下方的其他圖層上方。
- **層屬性。** 在此，可以使用數字控制項調整圖層的所有屬性。 首先選取圖層，然後調整其屬性。
- **複合** **URL。** UI底部是「複合URL」區域。 本教程的本節將不討論此問題，但在此您將看到模板被解構為一系列Image Serving URL修飾符。 此區域是可編輯的 — 如果您非常熟悉Image Server命令，則可以在此處手動編輯模板。 但你也能打破它。 與Photoshop一樣，層編號從0開始。 畫布是第0層，您自己添加的第一層是第1層。 混合模式確定圖層的像素如何與其下面的像素混合。 可以使用混合模式建立各種特殊效果。

#### 使用模板基礎編輯器

以下是啟動基本模板的工作流步驟：

1. 在Dynamic Media Classic，轉到 **生成>模板基礎**。 您可以不選擇任何內容，也可以從選擇影像開始，該影像將成為模板的第一層。
2. 選擇大小，然後按 **確定**。 此大小應與您在Photoshop設計的大小相匹配。 將載入模板編輯器。
3. 如果在步驟1中未選擇影像，請搜索或瀏覽左側資產面板中的影像並將其拖到工作區。

   - 影像將自動調整大小以適應畫布的大小。 如果您計畫替換高解析度影像，則通常會引入一個大（2000像素）的PTIFF影像並將其用作佔位符。
   - 這應是模板的最底層，但以後可以重新排序圖層。

4. 直接在工作區中調整圖層大小或重定點陣圖層，或通過調整「圖層屬性」面板中的設定。
5. 根據需要拖動其他影像圖層。 如果您也需要，請添加圖層效果。 查看主題 _添加圖層效果_&#x200B;的下界。
6. 按一下 **保存**，選擇一個位置，並為模板指定名稱。 您可以預覽，但此時模板將完全像平整的Photoshop影像 — 它尚未更改。

### 添加圖層效果

影像伺服器支援一些寫程式圖層效果 — 這些特殊效果會更改圖層內容的外觀。 它們在Photoshop的層效應也同樣有效。 它們附著到層，但與層無關地控制。 可以調整或移除它們，而無需對層本身進行永久更改。

- **投影**。 在圖層邊界外部應用陰影，該陰影由x和y像素偏移定位。
- **內陰影**。 在圖層邊界內應用陰影，通過x和y像素偏移定位。
- **外發光**。 將發光效果均勻地應用於圖層的所有邊緣。
- **內發光**。 均勻地在圖層的所有邊緣內應用發光效果。

![影像](assets/basic-templates/basic-templates-6.jpg)

_帶投影和不帶投影的圖層_

要添加效果，請按一下 **添加效果**，然後從菜單中選擇效果。 與普通圖層一樣，您可以在「圖層」面板中選擇效果，然後使用「圖層屬性」面板調整其設定。

陰影效果在水準或垂直方向遠離圖層時偏移，而「發光」效果在所有方向均勻應用。 內部效應作用在層的不透明部分的頂部，而外部效應僅影響透明區域。

瞭解有關[添加圖層效果](https://experienceleague.adobe.com/docs/dynamic-media-classic/using/template-basics/creating-template.html#using-shadow-and-glow-effects-on-layers)。

### 添加參數

如果您只是組合圖層並保存它們，則淨結果與平整的Photoshop影像無異。 使模板與眾不同的是，它能夠將參數添加到每個層的屬性中，以便通過URL動態更改這些參數。

在Dynamic Media Classic語中，參數是可連結到模板屬性的變數，因此可以通過URL操作它。 在將參數添加到圖層時，Dynamic Media Classic會在URL中顯示該屬性，方法是在參數名稱前加上美元符號($) — 例如，如果建立名為&quot;size&quot;的參數以更改圖層的大小，Dynamic Media Classic會更名您的參數$size。

如果不為屬性添加參數，則該屬性仍隱藏在Dynamic Media Classic資料庫中，且不顯示在URL中。

![影像](assets/basic-templates/parameters.png)

如果沒有參數，URL通常會長得多，特別是如果您也使用動態文本。 文本會在每個URL上添加許多額外字元。

最後，初始參數集將成為模板中屬性的預設值。 如果生成模板，添加參數，然後調用URL而不使用其參數，則Image Server將建立包含您保存在模板中的所有預設值的影像。 僅當要更改屬性時才需要參數。 如果屬性不需要更改，則不需要設定參數。

#### 建立參數

這是用於建立參數的工作流：

1. 按一下 **參數** 的子菜單。 「參數」(Parameters)螢幕開啟。 它列出圖層上的每個屬性及其值。
1. 選擇 **開** 選項。 將出現預設參數名稱。 您只能將參數添加到從其預設狀態更改的屬性。

   - 例如，如果添加一個圖層並將其保持在預設的xy位置0,0，則Dynamic Media Classic不會公開 **位置** 屬性。 要固定，請至少移動一個像素。 現在Dynamic Media Classic將揭露 **位置** 作為屬性，可以參數化。
   - 要將參數添加到show/hide屬性（該屬性可開啟和關閉圖層），請按一下 **顯示** 或 **隱藏圖層** 表徵圖以關閉圖層（如果需要，可以稍後重新開啟圖層）。 Dynamic Media Classic現在將揭露 **隱藏** 可參數化的屬性。

1. 將預設參數名稱更名為URL中更容易識別的名稱。 例如，如果要添加參數以更改影像頂部的橫幅圖層，請將預設名稱&quot;layer_2_src&quot;更改為&quot;banner&quot;。
1. 按 **關閉** 以退出「參數」螢幕。
1. 通過按一下 **參數** 按鈕並添加和更名參數。
1. 完成後保存更改。

>[!TIP]
>
>將參數更名為有意義的內容，並制定命名約定以規範這些名稱。 請確保設計和開發團隊事先同意命名約定。
>
>無法添加參數，因為您看不到該屬性？ 只需更改圖層的屬性（通過移動、調整大小、隱藏等）。 你現在應該看到那房產暴露了。

瞭解有關 [模板參數](https://experienceleague.adobe.com/docs/dynamic-media-classic/using/template-basics/creating-template-parameters.html)。

## 建立帶有文本層的模板

現在，您將學習如何建立包含文本層的基本模板。

### 瞭解動態文本

您現在知道如何使用影像圖層構建基本模板。 對於許多應用程式，您只需要這些。 正如您在上一練習中看到的，具有簡單文本（如「銷售」和「新建」）的圖層可柵格化並視為影像，因為它們的文本不需要更改。

但是，如果您需要：

- 添加標籤以說明「25%關閉」，其值為25%是可變的
- 添加影像頂部產品名稱的文本標籤
- 根據您看到模板的國家/地區，將您的層本地化為不同的語言

在這種情況下，您需要添加一些動態文本圖層，其中包含一些參數以控制文本和/或格式設定。

要建立文本，您需要上載一些字型 — 否則，Dynamic Media Classic將預設為Arial。 字型還必須發佈到影像伺服器，否則當字型嘗試呈現使用該字型的任何文本時，它將生成錯誤。

### RTF和文本參數

要使用「模板基礎知識」工具將變數添加到文本中，您應瞭解文本的呈現方式。 影像伺服器使用Adobe文本引擎生成文本，該引擎與Photoshop和Illustrator使用的引擎相同，並將其作為最終影像中的一個圖層複合。 要與引擎通信，影像伺服器使用RTF格式。

RTF是Microsoft為指定文檔格式而開發的檔案格式規範。 它是大多數文字處理和電子郵件軟體使用的標準標籤語言。 如果寫入URL &amp;text=\b1 Hello，則影像伺服器將生成帶有粗體文字「Hello」的影像，因為\b1是用於使文本加粗的RTF命令。

好消息是，Dynamic Media Classic為你製作了富文本。 無論何時在模板中鍵入文本並添加格式，Dynamic Media Classic都會悄悄地將RTF代碼自動寫入模板。 我們提到它的原因是您直接將參數添加到RTF本身，因此您對它有一點熟悉是很重要的。

#### 建立文本層

可以通過以下兩種方式在Dynamic Media Classic的模板中建立文本圖層：

1. 文本工具在Dynamic Media Classic。 我們將在下面討論這種方法。 「模板基礎知識」編輯器有一個工具，它允許您建立文本框、輸入文本和設定文本格式。 Dynamic Media Classic根據需要生成RTF並將其放在單獨的層中。
2. 提取文本（上載時）。 另一種方法是在Photoshop建立文本層，並將其另存為普通文本層（而不是將其柵格化為影像層）。 然後將檔案上傳到Dynamic Media Classic，然後使用 **提取文本** 的雙曲餘切值。 Dynamic Media Classic將使用RTF命令將每個Photoshop文本層轉換為Image Service文本層。 如果使用此方法，請確保先將字型上載到Dynamic Media Classic，否則Dynamic Media Classic將在上載時替換預設字型，並且無法簡單地重新替換正確的字型。

### 文本編輯器

使用文本編輯器輸入文本。 文本編輯器是WYSIWYG介面，允許您使用與Photoshop或Illustrator的格式控制項類似的格式控制項輸入和格式化文本。

![影像](assets/basic-templates/basic-templates-9.jpg)

_模板基礎知識文本編輯器。_

你將在 **預覽** 的子菜單。 還有 **源** 頁籤，如有必要，可手動編輯RTF。

一般工作流是 **預覽** 按鈕。

然後，選擇文本，並使用頂部的控制項選擇一些格式，如字型顏色、字型大小或對齊。 在文本按您想要的方式設定樣式後，按一下 **應用** 查看工作區預覽中的更新。 然後關閉文本編輯器以返回「模板基礎知識」主窗口。

#### 使用文本編輯器

以下是在「模板基礎知識」生成頁中添加文本的工作流步驟：

1. 按一下 **文本** 的子菜單。
2. 拖出要顯示文本的文本框。 「文本編輯器」(Text Editor)窗口將在模式窗口中開啟。 在後台，您將看到模板，但在編輯完文本之前，模板不可編輯。
3. 鍵入在首次載入模板時要顯示的示例文本。 例如，如果您正在為個性化電子郵件影像建立文本框，則您的文本可能會說「您好」。 現在是拯救的時候了！」 稍後，您將添加一個文本參數，以用您在URL上發送的值替換「名稱」。 在您按一下 **應用**。
4. 要格式化文本，請使用滑鼠拖動來選擇文本，然後在UI中選擇格式控制項。

   - 有許多格式設定選項。 最常見的有字型（面）、字型大小和字型顏色，以及左/中/右對齊。
   - 別忘了先選擇文本。 否則，將無法應用任何格式。
   - 要選擇其他字型，請確保選擇文本並開啟「字型」菜單。 編輯器將顯示上載到Dynamic Media Classic的所有字型的清單。 如果電腦上也安裝了字型，則字型將以黑色顯示。 如果電腦上未安裝，將以紅色顯示。 但是，當您按一下「預覽」(Preview)窗口時，它仍會呈現在 **應用**。 您只需將字型上傳到Dynamic Media Classic，即可讓任何使用Dynamic Media Classic的人使用。 發佈後，Image Server將使用這些字型來生成文本 — 您的用戶不需要安裝任何字型來查看您建立的文本，因為它是影像的一部分。
   - 與Photoshop和Illustrator不同，影像伺服器可以在文本框中垂直對齊文本。 預設為頂部對齊。 要更改此項，請選擇文本，然後選擇 **中間** 或 **底部** 從 **垂直對齊** 的子菜單。
   - 如果文本對框太大（或文本框太小），則全部或部分文本將被剪切並消失。 減小字型大小或增大框。

5. 按一下 **應用** 在工作區窗口中查看更改生效。 必須按一下 **應用**，否則您將丟失編輯內容。
6. 完成後，按一下 **關閉**。 如果要返回到編輯模式，請按兩下文本層以重新開啟文本編輯器。

文本編輯器將精確預覽字型的大小（如果在系統上本地安裝了字型）。

### 關於向文本圖層添加參數

現在，我們遵循添加文本參數的類似過程，就像添加圖層參數一樣。 文本層還可以根據尺寸、位置等取圖層參數；但是，它們可以採用附加參數，以便您控制RTF的任何方面。

與層參數不同，您只選擇要更改的值並向其中添加參數，而不是向整個屬性添加參數。

![影像](assets/basic-templates/basic-templates-10.jpg)

RTF示例：

![影像](assets/basic-templates/sample-rtf.png)

檢查RTF時，需要確定要更改的每個設定的位置。 在上面的RTF中，有些可能有意義，你可以看到格式的來源。

你可以看到&quot;巧克力薄荷沙&quot;這個詞就是文字本身

- 有對字型Poor Richard的引用 — 這是選擇字型的地方。
- 您可以看到RGB值：\red56\green53\blue4  — 這是文本顏色。
- 雖然字型大小為20，但您看不到數字20。 但是，您確實看到命令\fs40 — 出於一些奇數原因，RTF將字型測量為半點。 因此\fs40是字型大小！

您有足夠的資訊來建立參數，但是在Image Serving文檔中完整引用了所有RTF命令。 訪問 [影像服務文檔](https://experienceleague.adobe.com/docs/dynamic-media-developer-resources/image-serving-api/image-serving-api/http-protocol-reference/text-formatting/c-text-formatting.html#concept-0d3136db7f6f49668274541cd4b6364c)。

#### 向文本層添加參數

以下是向文本圖層添加參數的步驟。

1. 按一下 **參數** 按鈕(「P」)。 「參數」(Parameters)螢幕開啟。 的 **常用** 頁籤列出圖層上的每個屬性及其值。 在此可添加常規層參數。
1. 按一下 **文本** 頁籤。 這裡你可以看到頂部的RTF您添加的參數位於下面。
1. 要添加參數，請首先突出顯示要更改的值，然後按一下 **添加參數** 按鈕 確保只選擇命令的值，而不是整個命令本身。 例如，要在上面的示例RTF中為字型名稱設定參數，我只會突出顯示「Poor Richard」並添加一個參數，但不會同時添加「\f0」。 按一下 **添加參數** ，它顯示在下面的清單中，參數值在RTF中顯示為紅色，但仍處於選中狀態。 如果需要刪除參數，請按一下旁邊的複選框 **開** 關閉該參數，它就消失了。
1. 按一下將參數更名為更有意義的名稱。
1. 完成後，RTF將在參數存在時以綠色突出顯示，並在下面列出參數名稱和值。
1. 按一下 **關閉** 以退出「參數」螢幕。 然後按 **保存** ，以保存模板。 完成編輯後，按 **關閉** 退出「模板基礎知識」頁。
1. 按一下 **預覽** 在Dynamic Media Classictest模板。 要test文本參數，請在預覽窗口中鍵入新文本或新值。 要更改字型，必須鍵入字型的確切RTF名稱。

>[!TIP]
>
>要將參數添加到文本顏色，請分別添加紅色、綠色和藍色的參數。 例如，如果RTF `\red56\green53\blue46`，將為值56、53和46添加單獨的紅色、綠色和藍色參數。 在URL中，您可以通過調用以下三項來更改顏色： `&$red=56&$green=53&$blue=46`。

瞭解如何 [建立動態文本參數](https://experienceleague.adobe.com/docs/dynamic-media-classic/using/template-basics/creating-template-parameters.html#creating-dynamic-text-parameters)。

## 發佈和建立模板URL

### 建立影像預設

為模板建立預設不是必需步驟。 我們建議將其作為最佳做法，以便始終以1:1大小調用模板，同時還可向任何大影像圖層添加銳化功能，以便調整大小以適應模板。 如果調用沒有預設的影像，影像伺服器可以任意調整影像大小至預設大小（約400像素），並且不會應用預設銳化。

模板的「影像預設」沒有什麼特別之處。 如果已為大小相同的靜態影像設定了預設，則可以改用它。

### 發佈

您需要運行發佈才能查看將更改即時推送到映像伺服器。 請記住需要發佈的內容：各種影像資產層、動態文本的字型以及模板本身。 與Dynamic Media Classic的其他富媒體資產（如影像集和旋轉集）類似，基本模板是人工構造 — 它是資料庫中的一行項，使用一系列影像服務命令引用影像和字型。 因此，在發佈模板時，您所做的只是更新映像伺服器上的資料。

瞭解有關 [發佈模板](https://experienceleague.adobe.com/docs/dynamic-media-classic/using/template-basics/publishing-templates.html)。

### 模板URL構造

基本模板與前面所述的普通影像調用具有相同的基本URL語法。 模板通常會有更多修改量(用和號(&amp;)分隔的命令)，例如帶值的參數。 但主要區別在於您將模板作為主映像調用，而不是調用靜態映像。

![影像](assets/basic-templates/basic-templates-11.jpg)

與「影像預設」不同，參數在預設名稱的每側都有一個美元符號($)，在開頭有一個美元符號。 這些美元信號的放置是重要的。

**正確：**

`$text=46-inch LCD HDTV`

**錯誤：**

`$text$=46-inch LCD HDTV`

`$text=46-inch LCD HDTV$`

`text=46-inch LCD HDTV`

如前所述，參數用於更改模板。 如果調用模板時沒有參數，它將恢復為在「模板基礎知識」創作工具中設計的預設設定。 如果屬性不需要更改，則不需要設定參數。

![影像](assets/basic-templates/sandals-without-with-parameters.png)
_不設定參數（上）和參數（下）的模板示例。_
