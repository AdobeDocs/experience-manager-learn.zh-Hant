# 裁剪、調整影像和縮放目標 {#crop-adjusted-zoom-targets}

Dynamic Media Classic主映像概念的主要優勢之一是您可以重新調整映像資產的用途。 傳統上，您必須建立每個影像的單獨的裁剪版本以顯示詳細資訊或色板。 使用Dynamic Media Classic時，您可以在單個主伺服器上執行相同的任務，並將這些裁剪的版本另存為新的物理檔案或作為不佔用儲存空間的虛擬衍生檔案。

在本教程的本節結束時，您將瞭解如何：

- 在Dynamic Media Classic裁剪映像，並另存為新主檔案或虛擬映像。 [深入了解](https://experienceleague.adobe.com/docs/dynamic-media-classic/using/master-files/cropping-image.html).
- 保存虛擬調整後的映像，並使用它們代替主資產。 [深入了解](https://experienceleague.adobe.com/docs/dynamic-media-classic/using/master-files/adjusting-image.html).
- 在影像上建立縮放目標以顯示其高亮。 [深入了解](https://experienceleague.adobe.com/docs/dynamic-media-classic/using/zoom/creating-zoom-targets-guided-zoom.html).

## 裁切

Dynamic Media Classic在UI中提供了一些影像編輯工具，包括裁剪工具。 您可能希望在Dynamic Media Classic內裁剪主影像，原因有多。 例如：

- 您無權訪問原始檔案。 您希望使用不同的裁剪或長寬比來顯示影像，但您的電腦上沒有原始檔案，或者正在家庭中工作。 在這種情況下，您可以進入Dynamic Media Classic，查找影像，裁剪並保存影像，或將其另存為新版本。
- 刪除多餘的空白。 圖片被拍得太白，讓產品看起來很小。 您希望您的縮略圖影像盡可能填充畫布。
- 要建立調整後的映像，請建立不佔用磁碟空間的映像的虛擬副本。 有些公司有業務規則，要求它們保留同一映像的單獨副本，但名稱不同。 或者，您需要同一影像的裁剪和未裁剪版本。
- 從源影像生成新影像。 例如，您可能希望建立顏色色板或主影像的詳細資訊。 您可以在Adobe Photoshop執行此操作，單獨上傳或在Dynamic Media Classic使用裁剪工具。

>[!NOTE]
>
>以下關於裁剪的討論中的所有URL僅用於說明目的；它們不是即時連結。

### 使用裁剪工具

您可以從資產的「詳細資訊」頁面或按一下「Cont」(Dynamic Media Classic)中的「裁剪」工具 **編輯** 按鈕 可以使用工具以兩種方式裁剪：

- 拖動裁剪窗口的手柄或在「大小」(Size)框中鍵入值的預設裁剪模式。 瞭解如何 [手動裁剪](https://experienceleague.adobe.com/docs/dynamic-media-classic/using/master-files/cropping-image.html#select-an-area-to-crop)。
- 修剪。 使用此選項可通過計算與影像不匹配的像素數來刪除影像周圍的額外空白。 瞭解如何 [通過修剪裁剪](https://experienceleague.adobe.com/docs/dynamic-media-classic/using/master-files/cropping-image.html#crop-to-remove-white-space-around-an-image)。

### _手動裁剪_

保存手動裁剪的版本時，影像似乎會被永久裁剪；Dynamic Media Classic實際上是通過添加內部URL修飾符來裁剪影像來隱藏像素。 發佈時，影像被裁剪的情況將呈現給所有人，但您稍後可以返回「裁剪編輯器」並刪除裁剪。

然後，您可以選擇是另存為新主影像還是另存為主圖的附加視圖。 新主檔案是佔用儲存空間的新物理檔案(如TIFF或JPEG)。 附加視圖是不佔用伺服器空間的虛擬映像。 我們不建議您選擇「替換原稿」，因為這將覆蓋主檔案並使裁剪永久化。 如果另存為新主體或附加視圖，則必須選擇新資產ID。 與其他資產ID一樣，此名稱在Dynamic Media Classic必須唯一。

### _裁切裁剪_

如果在影像的主主題周圍上載包含過多空白（額外畫布）的影像，則調整大小後，該影像在Web上看起來會小得多。 對於150像素或更小的縮略圖，這一點尤為明顯 — 照片的主題可能會在周圍所有額外空間中丟失。

比較同一映像的這兩個版本。

![影像](assets/crop-adjusted-zoom-targets/trim-cropping.jpg)

通過移除產品周圍的額外空間，右邊的影像更加突出。 可以使用「裁剪」工具一次完成一個影像，或在上載時以批處理形式運行。 如果希望將所有映像始終裁切到主主題的邊界，建議以批處理方式運行。 修剪到邊界框 — 影像周圍的矩形。

>[!NOTE]
>
>修剪不會在影像周圍建立透明度。 為此，您需要在影像上嵌入剪輯路徑，並使用 **從剪輯路徑建立蒙版** 上載選項。
>
>此外，在使用 **保存** 選項，在「裁剪編輯器」螢幕中顯示影像，然後選擇 **重置** 按鈕

### _上載時裁剪_

如前所述，您還可以選擇在上載影像時裁剪影像。 要在上載時使用修剪，請按一下 **作業選項** 按鈕，然後在「裁剪選項」下選擇 **修剪**。

Dynamic Media Classic將記住下次上傳的此選項。 雖然您可能希望它裁剪影像以用於此上載，但您可能不希望每次上載都裁剪這些影像。 另一個選項是設定一個特殊的計畫FTP上載作業，並將裁剪選項放在那裡。 這樣，只有在需要裁剪影像時才會運行作業。

>[!IMPORTANT]
>
>如果為上傳設定裁剪，Dynamic Media Classic將在下次設定時添加cookie以記住該設定。 作為最佳做法，按一下 **重置為公司預設值** 按鈕，清除上次上載後剩餘的任何裁剪選項；否則，可能會意外裁剪下一批影像。

### 按URL裁剪

雖然在Dynamic Media Classic並不明顯，但您也可以僅通過URL（甚至向影像預設添加裁剪）進行裁剪。

每次使用「裁剪」工具時，都會在底部的欄位中看到URL值。 您可以取這些值，並將它們作為URL修飾符直接應用於影像。

![影像](assets/crop-adjusted-zoom-targets/cropping-by-url.png)
_裁剪編輯器底部的裁剪命令修飾符_

![影像](assets/crop-adjusted-zoom-targets/uncropped-cropped.png)

由於在使用裁切時必鬚根據每個影像計算大小，因此無法通過URL自動計算大小。 裁切只能在上載時運行，或通過一次應用一個影像來運行。

### _在影像預設中裁剪_

「影像預設」有一個欄位，您可以在其中添加額外的「影像服務」命令。 要將與上面相同的裁剪添加到「影像預設」中，請編輯預設並貼上或在「URL修改量」欄位中鍵入值，然後保存並發佈。

![影像](assets/crop-adjusted-zoom-targets/cropping-in-image-preset.jpg)
_將裁剪命令（或任何命令）添加到影像預設的URL修飾符中。_

此裁剪現在將是該影像預設的一部分，並且每次使用時都會自動應用。 當然，這種方法依賴於需要相同作物量的所有影像。 如果影像不是全部以相同方式拍攝，則此方法對您無效。

## 調整後的影像

使用「裁剪」工具時，您可以 **另存為主視圖**。 保存後，將建立一種新類型的Dynamic Media Classic資產 — 調整後的影像。 調整後的影像（也稱為導數）是虛擬影像。 它根本不是一幅圖片。它是物理主映像的資料庫引用（如別名或快捷方式）。

### 真實形象請站起來`?`

您能判斷哪個是主圖，哪個是調整後的影像嗎？

![影像](assets/crop-adjusted-zoom-targets/real-image-stand-up.png)

在查看Dynamic Media Classic並查看SBR_MAIN2的「調整後影像」資產類型之前，您不應該知道。

調整後的映像不使用磁碟空間，因為它只作為行項存在於資料庫中。 它還永久地與原始資產掛鈎；如果刪除原始影像，則還會刪除「已調整影像」。 它可以由整個未裁切的影像或僅由影像（裁剪）的一部分組成。

![影像](assets/crop-adjusted-zoom-targets/adjusted-image.png)

通常使用裁剪工具建立調整後的影像；但是，也可以使用其他影像編輯器（「調整」和「銳化」工具）建立它們。

調整後的映像需要唯一的資產ID。 發佈時（您必須像任何其他資產一樣發佈），它們充當任何其他影像，並按其資產ID在URL上調用。 在「詳細資訊」(Detail)頁面上，您可以在 **構建和衍生產品** 頁籤。

![影像](assets/crop-adjusted-zoom-targets/derivatives.jpg)
_主影像ASIAN_BR_MAIN的已調整視圖_

## 縮放目標

在 **編輯** 菜單和 **詳細資訊** 的子菜單。 它們允許您設定「熱點」，以突出縮放影像的特定促銷功能。 縮放查看器可提供影像頂部的詳細資訊以及您建立的短標籤，而不是通過裁剪大型母版來建立單獨的影像。

![影像](assets/crop-adjusted-zoom-targets/arm-with-watch.jpg)

由於縮放目標本質上是商品銷售功能，並且需要瞭解產品的銷售點，因此通常由公司商品銷售或產品團隊中的人員建立。

該過程非常簡單 — 按一下該特徵，給它描述性名稱，然後保存。 如果目標相似，則可以將目標從一個映像複製到另一個映像，但該過程是手動的。 在Dynamic Media Classic，無法自動建立縮放目標，因為每個影像都不同，並且具有不同的功能。

決定是否使用縮放目標的另一個因素是您選擇的查看器。 並非所有查看器類型都可以顯示「縮放目標」（例如，Fly-out查看器不支援它們）。

瞭解如何 [建立縮放目標](https://experienceleague.adobe.com/docs/dynamic-media-classic/using/zoom/creating-zoom-targets-guided-zoom.html#creating-and-editing-zoom-targets)。

![影像](assets/crop-adjusted-zoom-targets/zoom-targets.jpg)

### 使用縮放目標工具

這是在Dynamic Media Classic建立目標的工作流程。

1. 瀏覽到您的映像，按一下 **編輯** 按鈕 **縮放目標**。
2. 將載入縮放目標編輯器。 您會在中間看到您的影像，在頂部看到一些按鈕，在右側看到一個空的目標面板。 在左下角，您將看到選定的查看器預設。 預設值為「Zoom1-Guided」。
3. 用滑鼠移動紅色框，然後按一下以建立新目標。

   - 紅色框是目標區域。 當用戶按一下該目標時，它將放大到框內的區域。
   - 目標大小由「查看器預設」內的視圖大小確定。 這決定了主縮放影像的大小。 請參閱 _設定視圖大小_&#x200B;的下界。

4. 您將看到您剛剛建立的目標變為藍色，在右邊將看到該目標的縮略圖版本以及預設名稱&quot;target-0&quot;。
5. 要更名目標，請按一下其縮略圖，鍵入新的 **名稱**，然後按一下 **輸入** 或 **頁籤**  — 如果您按一下「離開」，則不會保存您的姓名。
6. 選中目標時，框周圍將有綠色虛線，您可以調整其大小並移動它。 拖動拐角以調整大小，或拖動目標框以移動它。

   - 這將在預設的「自定義」縮放查看器中載入影像。 確保「查看器預設」支援「縮放目標」 — 通常，所有標準預設都設計為與「縮放目標」一起使用。 要使用目標，請將滑鼠懸停在目標縮略圖（或熱點表徵圖）上以查看標籤，然後按一下該標籤以查看查看器放大到該功能。
   - 與您在Dynamic Media Classic所做的所有其他工作一樣，您必須發佈以讓「縮放目標」在Web上即時。 如果已使用支援目標的查看器，則這些查看器將立即顯示（一旦清除了快取）。 但是，如果您未使用啟用縮放目標的查看器，這些查看器將保持隱藏狀態。

      ![影像](assets/crop-adjusted-zoom-targets/zoom-target-green-box.jpg)

7. 此外，如果需要刪除目標，請按一下其縮略圖，然後按 **刪除目標** 按鈕或按鍵盤上的DELETE鍵。
8. 繼續按一下以添加新目標、更名和/或添加後調整大小。
9. 完成後，按一下 **保存** 按鈕，然後 **預覽**。

### 在縮放查看器預設中設定視圖大小

讓我們來談一談縮放目標的大小來自何處。 在縮放查看器的「查看器預設」中，有一個稱為視圖大小的設定。 視圖大小是查看器中縮放影像的大小。 它與舞台大小不同，舞台大小是查看器的總大小，包括UI元件和圖稿。

建立新目標時，它會從視圖大小導出其大小和縱橫比。 例如，如果視圖大小為200 x 200 ，則您將只能製作方形目標，最大縮放區域為200像素。 目標可以大於200像素，但始終為正方形。 但這也意味著縮放查看器內的影像只有200個像素 — 縮放目標的大小與查看器的大小有直接關係。 因此，在設定目標之前，您首先會決定查看器設計。

但是，預設情況下視圖大小為空（設定為0 x 0），因為主視圖影像的大小是動態的，並會根據舞台的大小自動派生。 問題是，如果您未在預設中明確設定視圖大小，則「縮放目標」工具將不知道生成目標的大小。

載入「縮放目標」工具時，視圖大小顯示在預設名稱旁邊。 比較內置的「縮放1指導」預設和自定義ZT_AUTHORING預設之間的視圖大小。

![影像](assets/crop-adjusted-zoom-targets/view-size.jpg)

您可以看到內置預設的大小為900 x 550，這意味著目標的大小永遠不可能小於這個較大的大小。 這可能太大了 — 如果你有2000像素的影像，你只能發出一個至少900像素的功能。 用戶可以手動進一步縮放，但是您無法引導它們更靠近。 將視圖大小設定為350 x 350允許目標非常接近地放大或調整大小。 但是，如果想在查看器中顯示更大的縮放影像，則需要建立新預設，因為您的預設鎖定在350像素。

### 建立或編輯支援縮放目標的查看器預設

要設定視圖大小，請建立或編輯支援縮放目標的查看器預設。

1. 在查看器預設中，轉到 **縮放設定** 的雙曲餘切值。
2. 設定寬度和高度。
3. 保存預設並關閉。 如果要在您的即時網站上使用該預設，您以後也必須發佈。
4. 進入「縮放目標」工具，選擇在左下角編輯的預設。 您將立即看到新視圖大小反映在目標中。
