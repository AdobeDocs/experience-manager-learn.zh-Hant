# 裁切、調整後的影像和縮放目標{#crop-adjusted-zoom-targets}

Dynamic Media古典主影像概念的主要優點之一，就是您可以將影像資產重新用於許多用途。 傳統上，您必須為每個影像建立個別的裁切版本，才能顯示細節或用於色票。 使用Dynamic Media經典版時，您可以對單一主版執行相同的工作，並將這些裁切的版本儲存為新的實體檔案，或儲存為不佔用儲存空間的虛擬衍生產品。

在本教學課程的本節結束前，您將瞭解如何：

- 在Dynamic Media經典影像中裁切影像，並另存為新的主檔案或虛擬影像。 [了解更多](https://docs.adobe.com/help/en/dynamic-media-classic/using/master-files/cropping-image.html).
- 儲存虛擬調整後的影像，並使用它們取代主資產。 [了解更多](https://docs.adobe.com/content/help/en/dynamic-media-classic/using/master-files/adjusting-image.html).
- 在影像上建立縮放目標，以顯示其反白顯示。 [了解更多](https://docs.adobe.com/content/help/en/dynamic-media-classic/using/zoom/creating-zoom-targets-guided-zoom.html).

## 裁切

Dynamic Media經典影像編輯工具在使用者介面中提供，包括裁切工具。 您可能想在Dynamic Media經典影像中裁切主影像，原因有很多。 例如：

- 您無法存取原始檔案。 您想要以不同的裁切或外觀比例來顯示影像，但是電腦上沒有原始檔案，或是在家中工作。 在這種情況下，您可以進入Dynamic Media經典影像、尋找影像、裁切並儲存，或儲存為新版本。
- 移除多餘的空白字元。 照片中的空白空間太大，產品看起來很小。 您希望縮圖影像能盡可能填滿畫布。
- 要建立調整後的映像，請建立不佔用磁碟空間的映像的虛擬拷貝。 有些公司有業務規則，要求他們保存相同影像的個別副本，但名稱不同。 或者，您可能想要相同影像的裁切和未裁切版本。
- 從來源影像建立新影像。 例如，您可能想要建立色票或主影像的細節。 您可以在Adobe Photoshop執行此動作，然後個別上傳或使用Dynamic Media經典中的裁切工具。

>[!NOTE]
>
>下列有關裁切的討論中的所有URL僅供說明之用；它們不是即時連結。

### 使用裁切工具

您可以從資產的「詳細資訊」頁面或按一下&#x200B;**編輯**&#x200B;按鈕訪問「Dynamic Media經典」中的裁切工具。 您可以使用工具以兩種方式裁切：

- 在預設裁切模式中，拖動裁切窗口的控點或在「大小」(Size)框中鍵入值。 瞭解如何「手動裁切」。[](https://docs.adobe.com/content/help/en/dynamic-media-classic/using/master-files/cropping-image.html#select-an-area-to-crop)
- 修剪。 使用此項來計算不符合影像的像素數，以移除影像周圍的額外空白。 瞭解如何透過修剪來裁切[。](https://docs.adobe.com/content/help/en/dynamic-media-classic/using/master-files/cropping-image.html#crop-to-remove-white-space-around-an-image)

### _手動裁切_

當您儲存手動裁切的版本時，影像會出現永久裁切；Dynamic Media·Classic實際上是透過新增內部URL修飾元來裁切影像來隱藏像素。 當您發佈時，每個人都會看到影像遭到裁切，不過您可以回到裁切編輯器，稍後再移除裁切。

然後，您可以選擇是另存為新主映像還是另存為主映像的附加視圖。 新的主檔案是佔用儲存空間的新實體檔案（例如TIFF或JPEG）。 另一個視圖是虛擬映像，它不佔用伺服器空間。 我們不建議您選擇「取代原稿」，因為這會覆寫您的主版並將裁切設為永久。 如果您儲存為新的主版或其他檢視，則必須選擇新的資產ID。 和其他資產ID一樣，這必須是Dynamic Media經典中的唯一名稱。

### _修剪裁切_

如果您在影像的主要主題周圍上傳包含太多空白（額外畫布）的影像，調整大小後，在網頁上的影像看起來會小得多。 對於150像素或更小的縮圖影像，尤其如此。 照片的主題可能會在周圍所有額外的空間里消失。

比較相同影像的這兩個版本。

![影像](assets/crop-adjusted-zoom-targets/trim-cropping.jpg)

移除產品周圍的額外空間，讓右側的影像更加醒目。 您可以使用裁切工具一次裁切一個影像，或在上傳時以批次處理方式執行。 如果您想要將所有影像一致地裁切至主主題的邊界，建議您以批次處理方式執行。 修剪裁切至邊界方框— 影像周圍的矩形。

>[!NOTE]
>
>修剪不會在影像周圍產生透明度。 為此，您需要在影像上嵌入剪裁路徑，並使用「從剪輯路徑建立遮色片」**上傳選項。**
>
>此外，若要在您使用&#x200B;**Save**&#x200B;選項裁切影像後，將影像還原為原始狀態，請在裁切編輯器畫面中顯示影像，並選取&#x200B;**Reset**&#x200B;按鈕。

### _上傳時裁切_

如前所述，您也可以選擇在上傳影像時裁切影像。 若要在上傳時使用修剪裁切，請按一下「作業選項」按鈕，然後在「裁切選項」下方選擇「修剪選項」。********

Dynamic Media·Classic會記住這個選項，以供下次上傳。 雖然您可能希望它裁切此上傳的影像，但您可能不希望每次上傳都裁切影像。 另一個選項是設定特殊的排程FTP上傳工作，並將裁切選項放在那裡。 如此，您只會在需要裁切影像時執行工作。

>[!IMPORTANT]
>
>如果您為上傳設定裁切，Dynamic Media經典會放置Cookie來記住下次設定。 在下次上傳之前，請按一下「重設為公司預設值&#x200B;**」按鈕，以清除上次上傳後留下的任何裁切選項；否則，您可能會意外裁切下一批影像。**

### 依URL裁切

雖然在Dynamic Media經典中並不明顯，但您也可以單純透過URL進行裁切（或甚至將裁切新增至影像預設集）。

每當您使用裁切工具時，底部的欄位中都會顯示URL值。 您可以擷取這些值，並直接將它們套用至影像做為URL修飾元。

### _在影像預設集中裁切_

「影像預設集」有一個欄位，您可以在其中新增額外的「影像伺服」指令。 若要將上述相同裁切新增至影像預設集，請編輯預設集並貼上或在「URL修飾元」欄位中輸入值，然後儲存並發佈。

![影](assets/crop-adjusted-zoom-targets/cropping-in-image-preset.jpg)

裁切現在會是該影像預設集的一部分，而且每次使用時都會自動套用。 當然，這種方法取決於所有需要相同裁切量的影像。 如果您的影像並非都以相同的方式拍攝，此方法將無法運作。

## 調整後的影像

使用「裁切」工具時，您可以選擇「另存為主版的附加視圖」。 ****&#x200B;在儲存後，這會創造出一種新型的Dynamic Media經典資產。 已調整的影像。 調整後的影像（也稱為衍生）是虛擬影像。 這其實根本不是一幅畫；它是物理主映像的資料庫參考（如別名或快捷方式）。

## 縮放目標

在影像的&#x200B;**Edit**&#x200B;功能表和&#x200B;**Details**&#x200B;頁面上也會找到縮放目標。 它們可讓您設定「作用點」，以反白標示縮放影像的特定銷售功能。 縮放檢視器不會裁切大型主影像，而是會在影像上提供詳細資料，以及您建立的短標籤。

![影像](assets/crop-adjusted-zoom-targets/arm-with-watch.jpg)

由於縮放目標本質上是銷售功能，而且需要瞭解產品的銷售點，因此通常是由公司銷售或產品團隊中的人員所建立。

這個過程非常簡單— 按一下功能，為其指定描述性名稱，然後儲存。 如果目標類似，則可將目標從一個映像複製到另一個映像，但該過程是手動的。 Dynamic Media經典影像無法自動建立縮放目標，因為每張影像都不同，而且功能也不同。

決定是否使用縮放目標的另一個因素是您選擇的檢視器。 並非所有檢視器類型都能顯示「縮放目標」（例如，「飛出」檢視器不支援這些類型）。

瞭解如何[建立縮放目標](https://docs.adobe.com/content/help/en/dynamic-media-classic/using/zoom/creating-zoom-targets-guided-zoom.html#creating-and-editing-zoom-targets)。

![影像](assets/crop-adjusted-zoom-targets/zoom-targets.jpg)

### 使用縮放目標工具

以下是在Dynamic Media經典中建立定位的工作流程。

1. 瀏覽至影像，按一下「編輯」按鈕，然後選擇「縮放目標」**「a3/」。******
2. 將載入縮放目標編輯器。 您會在中間看到您的影像，頂端有些按鈕，右邊有空白的目標面板。 在左下方，您會看到已選取的檢視器預設集。 預設值為&quot;Zoom1-Guided&quot;。
3. 用滑鼠移動紅色方塊，然後按一下以建立新目標。

   - 紅色方塊是目標區域。 當使用者按一下該目標時，會放大到方塊內的區域。
   - 目標大小由檢視器預設集內的檢視大小決定。 這會決定主縮放影像的大小。 請參閱下面的&#x200B;_設定視圖大小_。

4. 您會看到您剛建立的目標變成藍色，右邊會看到該目標的縮圖版本，以及預設名稱&quot;target-0&quot;。
5. 要更名目標，請按一下其縮略圖，鍵入新的&#x200B;**名稱** ，然後按一下&#x200B;**Enter**&#x200B;或&#x200B;**Tab** — 如果您按一下即可，您的名稱將不會儲存。
6. 選取目標時，框周圍會有綠色虛線，您可以調整大小並移動它。 拖曳轉角以調整大小，或拖曳目標方塊以移動。

   - 這會在預設的自訂縮放檢視器中載入影像。 確定檢視器預設集支援縮放目標— 一般而言，所有具有「-Guided」字詞的標準預設集都是專為搭配縮放目標而設計。 若要使用目標，請將滑鼠指標暫留在目標縮圖（或熱點圖示）上，以檢視標籤，然後按一下標籤即可檢視檢視該功能。
   - 就像您在Dynamic Media經典中做的其他所有作品一樣，您必須發佈，您的縮放目標才能在網路上即時顯示。 如果您已使用支援目標的檢視器，則會立即顯示（一旦清除快取）。 不過，如果您未使用啟用縮放目標的檢視器，這些檢視器將保持隱藏狀態。

      ![影像](assets/crop-adjusted-zoom-targets/zoom-target-green-box.jpg)

7. 此外，如果需要刪除目標，請按一下其縮略圖來選擇它，然後按&#x200B;**刪除目標**&#x200B;按鈕或按鍵盤上的DELETE鍵。
8. 繼續按一下以新增目標、重新命名及／或在新增後調整大小。
9. 完成後，按一下&#x200B;**保存**&#x200B;按鈕，然後按一下&#x200B;**預覽**。

### 在縮放檢視器預設集中設定檢視大小

讓我們來談談縮放目標的大小來源。 在縮放檢視器的檢視器預設集中，有一個稱為檢視大小的設定。 檢視大小是檢視器中縮放影像的大小。 它與舞台大小不同，舞台大小是檢視器的總大小，包括UI元件和圖稿。

當您建立新目標時，它會從檢視大小中衍生出其大小和外觀比例。 例如，如果您的檢視大小是200 x 200，則您只能製作正方形目標，最大縮放區域為200像素。 您的目標可以大於200像素，但一律為正方形。 但這也意味著，縮放檢視器內部的影像只有200像素。 縮放目標的大小與檢視器的大小有直接關係。 因此，您必須先決定檢視器設計，再設定目標。

但是，視圖大小預設為空白（設為0 x 0），因為主視圖影像的大小是動態的，並會根據舞台的大小自動衍生出來。 問題是，如果您未在預設中明確設定檢視大小，「縮放目標」工具將不知道要建立目標的大小。

載入「縮放目標」工具時，檢視大小會顯示在預設集名稱旁。 比較內建Zoom1引導預設集和自訂ZT_AUTHORING預設集之間的檢視大小。

![影像](assets/crop-adjusted-zoom-targets/view-size.jpg)

您可以看到內建預設集的大小為900 x 550，這表示目標永遠不會小於這麼大的大小。 這可能太大— 如果您有2000像素的影像，則只能呼叫至少900像素的功能。 使用者可以手動進一步縮放，但您無法進一步引導他們。 將檢視大小設為350 x 350可讓目標放大或放大。 但是，如果您想要在檢視器中放大縮放影像，則需要建立新的預設集，因為您的預設集鎖定在350像素。

### 建立或編輯支援縮放目標的檢視器預設集

若要設定檢視大小，請建立或編輯支援縮放目標的檢視器預設集。

1. 在檢視器預設集中，移至&#x200B;**縮放設定**&#x200B;選項。
2. 設定寬度和高度。
3. 儲存預設集，然後關閉。 如果您想要在即時網站上使用該預設集，您稍後也必須發佈。
4. 進入「縮放目標」工具，然後選擇您在左下角編輯的預設集。 您會立即看到新的檢視大小反映在您的目標中。
