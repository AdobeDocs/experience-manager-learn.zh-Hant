# 影片概觀 {#video-overview}

Dynamic Media Classic隨附自動轉換上傳視訊、視訊串流至桌上型電腦和行動裝置，以及根據裝置和頻寬最佳化的最適化視訊集。 視訊最重要的一點是，工作流程很簡單： 它的設計讓任何人都能使用它，即使他們不太熟悉視訊技術。

在本教學課程的本節結束前，您將瞭解如何：

- 將視訊上傳並編碼（轉碼）為不同的大小和格式
- 從可用的視訊預設集中選擇以進行上傳
- 新增或編輯視訊編碼預設集
- 在視訊檢視器中預覽視訊
- 將視訊部署至網頁和行動網站
- 在視訊中新增標題和章節標籤
- 為桌上型電腦和行動裝置使用者自訂和發佈視訊檢視器

>[!NOTE]
>
>本章中的所有URL僅用於說明目的；它們不是即時連結。

## Dynamic Media Classic視訊概觀

首先，讓我們使用Dynamic Media Classic更清楚瞭解視訊的可能性。

### 功能

Dynamic Media Classic視訊平台提供視訊解決方案的所有部分： 視訊的上傳、轉換和管理；在視訊中加入標題和章節標籤；以及使用預設集以輕鬆播放的功能。

它可讓您輕鬆發佈高品質的最適化視訊，以便在多種螢幕上串流，包括桌上型電腦、iOS、Android、Blackberry和Windows行動裝置。 「最適化視訊集」會針對以不同位元速率和格式（例如400 kbps、800 kbps和1000 kbps）編碼的相同視訊版本分組。 桌上型電腦或行動裝置會偵測可用頻寬。

此外，如果桌上型電腦或行動裝置的網路狀況有所改變，視訊品質會自動動態切換。 此外，如果客戶進入桌上型電腦的全螢幕模式，最適化視訊集會使用更佳的解析度來回應，進而改善客戶的觀看體驗。 使用最適化視訊集，讓客戶在多種螢幕和裝置上播放Dynamic Media Classic視訊時，可盡情播放。

### 視訊管理

處理視訊比處理靜態數位影像更複雜。 使用視訊，您可處理多種格式和標準，以及您的觀眾是否能播放視訊片段的不確定性。 Dynamic Media Classic可讓您輕鬆處理視訊，提供許多功能強大的工具，讓您輕鬆使用視訊，不但不需要複雜的工作。

Dynamic Media Classic可識別並可使用許多不同的來源格式。 然而，閱讀視頻只是努力的一部分。 您也必須將視訊轉換為適用於網頁的格式。 Dynamic Media Classic可讓您將視訊轉換為H.264視訊，以解決此問題。

使用許多專業和發燒友工具，自行轉換視訊變得十分複雜。 Dynamic Media Classic提供簡單易用的預設集，可針對不同的品質設定進行最佳化。 不過，如果您想要更自訂的項目，也可以建立您自己的預設集。

如果您有大量視訊，您會很欣賞Dynamic Media Classic中管理所有資產以及影像和其他媒體的能力。 您可以透過強穩的XMP中繼資料支援，來組織、編目和搜尋您的資產，包括視訊資產。

### 視訊播放

類似於將視訊轉換為方便Web使用且可存取的問題，是在您的網站上實作和部署視訊的問題。 選擇是購買播放器還是自行建立播放器，讓它與各種裝置和螢幕相容，然後維護您的播放器就是全職工作。

同樣地，Dynamic Media Classic的方法是讓您選擇符合您需求的預設集和檢視器。 您有許多不同的檢視器選擇，並有許多預設集的資料庫。

您可輕鬆將視訊發佈至網頁和行動裝置，因為Dynamic Media Classic支援HTML5視訊，這表示您可以鎖定執行各種瀏覽器的使用者，以及Android和iOS平台使用者。 串流視訊可讓您順暢地播放較長或高解析度的內容，而漸進式HTML5視訊則具備針對小螢幕最佳化的預設集。

視訊的檢視器預設集可視檢視器類型而部分設定。

和所有檢視器一樣，整合是透過單一Dynamic Media Classic URL（每個檢視器或視訊）進行。

>[!NOTE]
>
>最佳實務是使用Dynamic Media Classic HTML5視訊檢視器。 HTML5視訊檢視器中使用的預設集是強穩的視訊播放器。 結合單一播放器，即可設計使用HTML5和CSS的播放元件、內嵌播放，並視瀏覽器的功能使用可調式和漸進式串流，讓您將豐富型媒體內容的觸及面擴展至案頭、平板電腦和行動裝置使用者，並確保簡化視訊體驗。

關於Dynamic Media Classic視訊的最後一個附註可能適用於部分客戶：並非所有公司都會為其帳戶啟用自動轉換、串流或視訊預設集。 如果由於某些原因，您無法存取串流視訊的URL，可能就是這個原因。 您仍可上傳和發佈漸進式下載的視訊，並可存取所有視訊檢視器。 不過，若要運用完整的Dynamic Media Classic視訊功能，您必須聯絡客戶經理或銷售經理，以啟用這些功能。

進一步瞭解 [Dynamic Media Classic中的視訊](https://docs.adobe.com/content/help/en/dynamic-media-classic/using/video/quick-start-video.html)。

## Video 101

### 基本視訊概念和術語

在我們開始之前，讓我們先討論一些您應該熟悉的術語，以便使用視訊。 這些概念並非Dynamic Media Classic的特定概念，如果您要管理專業網站的視訊，您最好進一步瞭解相關主題。 在本節末尾，我們將推薦一些資源。

- **編碼／轉碼。** 編碼是套用視訊壓縮的程式，將原始、未壓縮的視訊資料轉換為格式，讓您更容易使用。 轉碼雖然類似，但是指從一種編碼方法轉換為另一種方法。

   - 使用視訊編輯軟體建立的主視訊檔案通常太大，而且格式不適合傳送至線上目的地。 它們通常會經過編碼，以便在桌上型電腦上快速播放和編輯，但無法透過網路傳送。
   - 為了將數位視訊轉換為適當的格式和規格，以便在不同螢幕上播放，視訊檔案會轉碼為較小、有效率的檔案大小，最適合傳送至網頁和行動裝置。

- **視訊壓縮。** 減少用於表示數字視頻影像的資料量，是空間影像壓縮和時間運動補償的組合。

   - 大部分的壓縮技術都有損耗，這意味著它們會捨棄資料，以實現較小的大小。
   - 例如，DV視訊壓縮量相對較小，可讓您輕鬆編輯來源視訊素材，但是太大，無法透過網路使用，甚至無法放入DVD。

- **檔案格式。** 格式是容器，類似於ZIP檔案，可決定在視訊檔案中組織檔案的方式，但通常不決定檔案的編碼方式。

   - 來源視訊的常用檔案格式包括Windows Media(WMV)、QuickTime(MOV)、Microsoft AVI和MPEG等。 Dynamic Media Classic發佈的格式為MP4。
   - 視訊檔案通常包含多首音軌。 視訊音軌（無音訊）和一或多個音軌（無視訊）— 是相互關聯，同步的。
   - 視訊檔案格式決定這些不同資料追蹤和中繼資料的組織方式。

- **轉碼器.** 視頻編解碼器描述通過使用壓縮對視頻進行編碼的算法。 音訊也透過音訊轉碼器進行編碼。

   - 轉碼器可將播放視訊所需的資訊量降到最低。 與每個單獨幀的資訊不同，只儲存關於一個幀和下一個幀之間差異的資訊。
   - 由於大部分視訊在一個影格之間幾乎沒有變化，所以轉碼器可提供高壓縮率，而檔案大小會變小。
   - 視頻播放器根據視頻的編解碼器解碼視頻，然後在螢幕上顯示一系列影像或幀。
   - 常見的視訊轉碼器包括H.264、On2 VP6和H.263。

![影像](assets/video-overview/bird-video.png)

- **解析度.** 視訊的高度和寬度（以像素為單位）。

   - 來源視訊的大小由相機決定，並從編輯軟體輸出。 HD相機通常會建立高解析度1920 x 1080視訊，但若要在網路上順暢播放，您會將它縮減取樣（調整大小）至較小的解析度，例如1280 x 720、640 x 480或更小。
   - 解析度會直接影響檔案大小以及播放該視訊所需的頻寬。

- **顯示外觀比例。** 視訊寬度與視訊高度的比例。 當視訊的長寬比不符合播放器的比例時，您可能會看到「黑條」或空白。 用於顯示視訊的兩個常見外觀比例是：

   - 4:3 (1.33:1). 用於幾乎所有標準解析度的電視廣播內容。
   - 16:9 (1.78:1). 用於幾乎所有的寬螢幕、高解析度電視內容(HDTV)和影片。

- **位元速率／資料速率。** 編碼為構成視訊播放單秒的資料量（每秒千位）。

   - 一般而言，位元速率越低，Web就越理想，因為下載速度越快。 但是，這也意味著由於壓縮損失，質量較低。
   - 良好的編解碼器應兼顧低比特率和高質量。

- **幀速率（每秒幀數，或FPS）。** 每秒視訊的影格數或靜止影像數。 通常，北美電視(NTSC)以29.97 FPS的速率播出；歐洲和亞洲電視(PAL)以25 FPS播出；而電影（模擬和數位）通常採用24(23.976)FPS.

   - 為了讓事情更加混亂，還有逐行和交錯的影格。 每個逐行幀包含整個影像幀，而隔行掃描幀包含影像幀中的每一行像素。 然後，影格會迅速播放，並呈現為混合在一起。 影片使用漸進式掃描方法，而數位視訊通常是隔行掃描。
   - 一般來說，不管源視頻素材是否隔行掃描，都無關緊要。 Dynamic Media Classic將保留轉換後視訊中的掃描方法。
   - 串流／漸進式傳送。 視訊串流是在連續串流中傳送媒體，當它送達時即可播放，而漸進式下載視訊會像從伺服器下載其他檔案一樣，在瀏覽器本機快取。

希望本入門簡介可協助您瞭解使用Dynamic Media Classic視訊的各種選項。

## 視訊工作流程

在Dynamic Media Classic中處理視訊時，您會遵循與處理影像類似的基本工作流程。

![影像](assets/video-overview/video-overview-2.png)

1. 首先，將視訊檔案上傳至Dynamic Media Classic。 若要這麼做，請開啟Dynamic Media Classic擴充功能面板底部的「工具選單 **」，然後選擇「上傳至Dynamic Media Classic >檔案至檔案夾名稱** 」，或「上傳至Dynamic Media Classic > **檔案夾至檔案夾名稱」******。 「檔案夾名稱」將是您目前瀏覽的副檔名為「檔案夾名稱」的檔案夾。 視訊檔案可能很大，因此我們建議使用FTP上傳大型檔案。 在上傳時，選擇一或多個視訊預設集來編碼您的視訊。 視訊可在上傳時轉碼為MP4視訊。 如需使用和建立編碼預設集的詳細資訊，請參閱下方的視訊預設集主題。 瞭解上傳 [和編碼視訊](https://docs.adobe.com/content/help/en/dynamic-media-classic/using/video/uploading-encoding-videos.html)。
2. 選取或選取並修改視訊檢視器預設集，並預覽您的視訊。 您可以選擇預先建立的檢視器預設集，或自訂您自己的檢視器預設集。 如果您要鎖定行動使用者，您不必在這裡做任何動作，因為行動平台不需要檢視器或預設集。 進一步瞭解在視 [訊檢視器中預覽視訊](https://docs.adobe.com/content/help/en/dynamic-media-classic/using/video/previewing-videos-video-viewer.html) , [以及新增或編輯視訊檢視器預設集](https://docs.adobe.com/content/help/en/dynamic-media-classic/using/video/previewing-videos-video-viewer.html#adding-or-editing-a-video-viewer-preset)。
3. 執行「視訊發佈」、取得URL並整合。 視訊工作流程與影像工作流程的這個步驟主要不同之處，在於您將執行特殊的視訊發佈，而非（或者可能也是）標準的影像伺服發佈。 案頭上的視訊檢視器整合功能與影像檢視器整合功能完全相同，但行動裝置的整合功能則更簡單： 您只需要視訊本身的URL。

### 關於轉碼

轉碼的定義早先是將一種編碼方式轉換為另一種編碼方式的程式。 在Dynamic Media Classic中，是將來源視訊從目前格式轉換為MP4的程式。 在視訊出現在案頭瀏覽器或行動裝置上之前，這是必要項。

Dynamic Media Classic可為您處理所有轉碼作業，這是一大優點。 您可以自行轉碼視訊並上傳已轉換為MP4的檔案，但是這個程式很複雜，需要複雜的軟體。 除非您知道自己在做什麼，否則您通常不會在第一次嘗試時獲得良好結果。

Dynamic Media Classic不僅可為您轉換檔案，還提供簡單易用的預設集。 您確實不需要對此程式的技術層面瞭解太多— 您應該知道的，大致是您想要從系統中移出的最終大小，以及您的最終用戶擁有的頻寬。

雖然預先建立的預設集十分方便，而且可滿足大部分需求，但有時您需要更自訂的工具。 在這種情況下，您可以建立自己的編碼預設集。 在Dynamic Media Classic中，編碼預設稱為視訊預設。 本章稍後將對此進行說明。

### 關於串流

另一個值得注意的主要功能是視訊串流，這是Dynamic Media Classic視訊平台的標準功能。 串流媒體在傳送時，會不斷由使用者接收並呈現給使用者。 這是重要的，也是可取的，原因很多。

串流通常需要的頻寬比漸進式下載要少，因為只有實際傳送的視訊部分會受到監視。 Dynamic Media Classic視訊串流伺服器和檢視器使用自動頻寬偵測，為使用者的網際網路連線提供最佳的串流。

透過串流，視訊的播放速度會比使用其他方法來得快。 此外，它還能更有效率地使用網路資源，因為只會將觀看的視訊部分傳送至用戶端。

另一種傳送方式是漸進式下載。 與串流視訊相比，漸進式下載只有一個一貫的優點： 您不需要串流伺服器就能傳送視訊。 當然，Dynamic Media Classic就是在這裡推出的。 Dynamic Media Classic有內建在平台中的串流伺服器，因此您不需要維護此專用硬體的麻煩或額外成本。

漸進式下載視訊可從任何一般的網路伺服器提供。 雖然這可以方便而且具成本效益，但請記住漸進式下載的搜尋和導覽功能有限，而且使用者可以存取和重複使用您的內容。 在某些情況下，例如在非常嚴格的網路防火牆後播放，流傳輸可能會被阻止；在這些情況下，可以要求回滾到漸進式交付。

漸進式下載是愛好者或具備低流量需求的網站的好選擇；如果他們不介意自己的內容是否被快取在使用者的電腦上；如果他們只需要提供較短長度的視訊（不到10分鐘）;或是訪客因故無法接收串流視訊。

如果您需要進階功能並控制視訊傳送，以及／或您需要向更廣大的觀眾（例如數百個同時檢視的觀眾）顯示視訊、追蹤和報告使用情況或檢視統計資料，或想要為您的觀眾提供最佳的互動播放體驗，您就需要將視訊串流化。

最後，如果您擔心因智慧財產或版權管理問題而保護您的媒體，串流會提供更安全的視訊傳送，因為當串流時，媒體不會儲存至用戶端的快取。

## 視訊預設集

上傳視訊時，您可從一或多個預設集中選擇，這些預設集包含透過編碼將主視訊轉換為適用於網頁的格式的設定。 視訊預設集有兩種類型：最適化視訊預設集和單一編碼預設集。

請參閱 [可用的視訊預設集](https://docs.adobe.com/content/help/en/dynamic-media-classic/using/setup/application-setup.html#video-presets-for-encoding-video-files)。

「最適化視訊預設集」預設會啟動，這表示可進行編碼。 如果您想要使用單一編碼預設集，管理員將需要啟用它，才能將它顯示在視訊預設集清單中。

瞭解如何啟 [用或取消啟用視訊預設集](https://docs.adobe.com/content/help/en/dynamic-media-classic/using/video/uploading-encoding-videos.html#activating-or-deactivating-video-encoding-presets)。

您可以選擇Dynamic Media Classic隨附的許多預先建立預設集，或自行建立預設集；不過，預設不會選取要上傳的預設集。 換言之，如果您 **未在上傳時選取「視訊預設集」，您的視訊將無法轉換，而且可能無法發佈**。 不過，您可以自行離線轉換視訊，並上傳和發佈。 只有當您要Dynamic Media Classic為您進行轉換時，才需要視訊預設集。

上傳時，您可以在「工作選項」面板中選擇「 **視訊選項** 」來選取視訊預設集。 然後，您可以選擇是否要為電腦、行動裝置或平板電腦編碼。

- 電腦是桌上型電腦。 在這裡，您通常會找到耗用更多頻寬的較大預設集（例如HD）。
- 行動裝置和平板電腦為iPhone和Android智慧型手機等裝置建立MP4視訊。 行動裝置和平板電腦的唯一差異在於，平板電腦的預設集通常具有較高的頻寬，因為它們是以WiFi使用量為基礎。 行動裝置預設集已最佳化，以降低3G使用速度。

### 在選擇預設集之前先自問的問題

在選擇預設集時，您應瞭解您的觀眾以及來源素材。 您對客戶有何瞭解？ 他們如何觀看視頻。 使用電腦螢幕或行動裝置？

您的視訊解析度為何？ 如果您選擇比原稿大的預設集，則可能會產生模糊／像素化視訊。 如果您的視訊大於預設集，但不要選擇大於來源視訊的預設集，則可以。

它的長寬比是多少？ 如果您在轉換的視訊周圍看到黑條，則您會選擇錯誤的外觀比例。 Dynamic Media Classic無法自動偵測這些設定，因為它必須先檢查檔案，才能上傳。

### 視訊選項劃分

「視訊預設集」會指定這些設定，以決定視訊的編碼方式。 如果您不熟悉這些術語，請參閱上述的基本視訊概念和術語。

![影像](assets/video-overview/video-overview-3.jpg)

- **外觀比例.** 通常為標準4:3或寬螢幕16:9。
- **大小.** 這與顯示解析度相同，並以像素為單位測量。 這與長寬比有關。 以16:9的比例計算，視訊會是432 x 240像素，而4:3的比例會是320 x 240像素。
- **FPS.** 標準影格速率為每秒30、25或24個影格(fps)，視視訊標準而定。 NTSC、PAL或Film。 此設定並不重要，因為Dynamic Media Classic會一律使用與來源視訊相同的影格速率。
- **格式.** 這是MP4。
- **頻寬。** 這是您目標使用者所需的連線速度。 他們的網際網路連線是快還是慢？ 他們通常使用桌上型電腦或行動裝置嗎？ 這也與解析度（大小）有關，因為視訊愈大，所需的頻寬愈多。

### 判斷視訊的資料速率或「位元速率」

計算視訊的位元速率是將視訊提供至網頁時最不為人所知的因素之一，但可能最重要的因素，因為它會直接影響使用者體驗。 如果您設定的位元速率過高，您的視訊品質會很高，但效能卻不佳。 當視訊播放時，網際網路連線較慢的使用者將被迫等候視訊不斷暫停。 但是，如果設定得太低，品質就會受到影響。 在「視訊預設集」中，Dynamic Media Classic會根據您的目標頻寬，建議一系列資料。 這是一個很好的起點。

不過，如果您想自己弄清楚，則需要一部位元速率計算器。 這是視訊專業人員和愛好者常用的工具，可用來估計特定串流或媒體（例如DVD）中的資料量。

## 建立自訂視訊預設集

有時，您可能會發現需要特殊的「視訊預設集」，而不符合內建編碼視訊預設集的設定。 如果您有特定大小的自訂視訊，例如從3D動畫軟體建立的視訊，或是從原始大小裁切的視訊，可能會發生這種情況。 您可能想要嘗試不同的頻寬設定，以提供更高品質或更低品質的視訊。 無論如何，您都需要建立自訂的單一編碼視訊預設集。

### 視訊預設集工作流程

1. 「視訊預設集」位於「設 **定>應用程式設定>視訊預設集」下**。 您會在這裡找到公司所有可用的編碼預設集清單。

   - 每個串流視訊帳戶都有數十種預設集，如果您建立自己的自訂預設集，您也會在這裡看到這些預設集。
   - 您可以使用下拉式選單依類型篩選。 預設集分為電腦、行動裝置和平板電腦。
      ![影像](assets/video-overview/video-overview-4.jpg)

2. 「作用中」欄可讓您選擇要在上傳時顯示所有預設集，還是只顯示您選擇的預設集。 如果您在美國，您可能想要取消勾選「歐洲PAL」預設集，如果在英國／歐洲、中東和非洲地區，請取消勾選「NTSC預設集」。
3. 按一下「 **新增** 」按鈕以建立自訂預設集。 這會開啟「新增視訊預設集」面板。 此處的程式類似於建立影像預設集。
4. 首先，為預設集 **指定「預設集名稱** 」，以顯示在預設集清單中。 在上述範例中，此預設集是用於畫面擷取教學課程影片。
5. 「說 **明** 」是選用的，但會提供您的使用者說明此預設集的用途的工具提示。
6. 「 **編碼檔案字尾** 」會附加至您在此建立之視訊名稱的結尾。 請記住，您將擁有主視訊以及此編碼視訊，這是主視訊的衍生項目，而且Dynamic Media Classic中沒有兩個資產可以擁有相同的資產ID。
7. **播放裝置** ，是您選擇所要視訊檔案格式（電腦、行動裝置或平板電腦）的位置。 請記住，Mobile和Tablet會產生相同的MP4格式。 Dynamic Media Classic只需要知道要放置預設集的類別；不過，理論上的不同在於，平板電腦的預設集通常是為了更快速的網際網路連線，因為所有預設集都支援WiFi。
8. **「目標資料率** 」是您必須親自瞭解的，不過您可以在下圖上看到建議的範圍。 您也可以將滑桿拖曳至近似的目標頻寬。 如需更精確的圖形，請使用位元速率計算器。 有一些嘗試和錯誤。

   ![影像](assets/video-overview/video-overview-5.jpg)

9. 設定源檔案的外 **觀比例**。 此設定會直接系結至下方的大小。 如果您選擇「 _自訂_」，則必須手動輸入寬度和高度。
10. 如果您選擇外觀比例，則為「解析度大小 **** 」設定一個值，Dynamic Media Classic會自動填入另一個值。 不過，對於自訂外觀比例，請填入這兩個值。 您的大小應符合您的資料速率。 如果您設定極低的資料速率和較大的大小，您會預期品質會很差。
11. 按一 **下「儲存** 」以儲存預設集。 與其他所有預設不同，您目前不需要發佈，因為預設集僅用於上傳檔案。 稍後，您必須發佈已編碼的視訊，但預設集僅供內部Dynamic Media Classic使用。
12. 若要確認您的視訊預設集位於上傳清單中，請前往「 **上傳**」。選擇「 **工作選項** 」並展開「 **視訊選項」**。 您的預設集會列在您選擇的播放裝置（電腦、行動裝置或平板電腦）的類別中。

進一步瞭解 [新增或編輯視訊預設集](https://docs.adobe.com/content/help/en/dynamic-media-classic/using/video/uploading-encoding-videos.html#adding-or-editing-a-video-encoding-preset)。

## 視訊搜尋引擎最佳化

搜尋引擎最佳化(SEO)是改善搜尋引擎中網站或網頁可見度的程式。 雖然搜尋引擎擅長收集有關文字內容的資訊，但是無法充份取得視訊的資訊，除非已提供這些資訊給他們。 使用Dynamic Media Classic視訊搜尋引擎優化(SEO)，您可以使用中繼資料來提供您視訊的說明。 「視訊搜尋引擎最佳化」功能可讓您建立視訊網站地圖和媒體RSS(mRSS)饋送。

- **視訊網站地圖**。 確切告知Google視訊內容在網站上的位置和內容。 因此，視訊可在Google上完全搜尋。 例如，視訊網站地圖可以指定視訊的執行時間和類別。
- **mRSS摘要**。 內容發佈者用來將媒體檔案饋送至Yahoo! 視訊搜尋。 Google支援視訊網站地圖和媒體RSS(mRSS)饋送通訊協定，以將資訊送出至搜尋引擎。

當您建立視訊網站地圖和mRSS饋送時，會決定要包含哪些視訊檔案的中繼資料欄位。 如此，您就可將視訊描述給搜尋引擎，讓搜尋引擎能夠更精確地將流量導向網站上的視訊。

建立網站地圖或動態消息後，您就可讓Dynamic Media Classic自動發佈、手動發佈，或只是產生檔案以供日後編輯。 此外，Dynamic Media Classic每天都可自動產生並發佈此檔案。

在程式結束時，您會將檔案或URL送出至您的搜尋引擎。 這項工作是在Dynamic Media Classic外完成；不過，我們將在下面簡要討論。

### 網站地圖/mRSS檔案的需求

為了讓Google和其他搜尋引擎不會拒絕您的檔案，它們必須採用適當的格式並包含特定資訊。 Dynamic Media Classic將產生格式正確的檔案；但是，如果某些視訊無法取得這些資訊，則檔案中將不會包含這些資訊。

必填欄位包括「著陸頁面」（提供視訊的頁面URL，而非視訊本身的URL）、「標題」和「說明」。 每個視訊都必須有這些項目的項目，否則不會納入產生的檔案。 可選欄位為「標籤」和「類別」。

還有兩個必填欄位— 內容URL、視訊資產本身的URL，以及視訊縮圖影像的URL縮圖。 不過，Dynamic Media Classic會自動為您填入這些值。

建議的工作流程是使用XMP中繼資料在上傳之前，先將此資料內嵌在您的視訊中，Dynamic Media Classic會在上傳時加以擷取。 您會使用Adobe Bridge等應用程式。 它隨附於所有Adobe Creative Cloud應用程式中— 來填入標準中繼資料欄位。

遵循此方法，您就不需要使用Dynamic Media Classic手動輸入此資料。 不過，您也可以使用Dynamic Media Classic中的中繼資料預設集，快速輸入相同的資料。

有關該主題的詳細資訊，請參 [閱查看、添加和導出元資料](https://docs.adobe.com/content/help/en/dynamic-media-classic/using/managing-assets/viewing-adding-exporting-metadata.html)。

![影像](assets/video-overview/video-overview-7.jpg)

在填入中繼資料後，您就可以在該視訊資產的「詳細資料檢視」中看到它。 關鍵字也可能存在，但它們位於「關鍵字」標籤下。

- 進一步瞭解新 [增關鍵字](https://docs.adobe.com/content/help/en/dynamic-media-classic/using/managing-assets/viewing-adding-exporting-metadata.html#add-or-edit-keywords)。
- 進一步瞭解 [視訊搜尋引擎](https://docs.adobe.com/content/help/en/dynamic-media-classic/using/setup/video-seo-search-engine-optimization.html)。
- 瞭解視 [訊搜尋引擎最佳化的設定](https://docs.adobe.com/content/help/en/dynamic-media-classic/using/setup/video-seo-search-engine-optimization.html#choosing-video-seo-settings)。

#### 設定視訊搜尋引擎最佳化(SEO)

設定視訊搜尋引擎最佳化(SEO)從選擇您想要的格式類型、產生方法，以及應將哪些中繼資料欄位放入檔案開始。

1. 前往「設 **定>應用程式設定>視訊搜尋引擎最佳化>設定」**。
2. 在「生 **成模式** 」菜單中，選擇檔案格式。 預設為「關閉」，因此若要啟用它，請選擇「視訊網站地圖」、「mRSS」或「兩者皆是」。
3. 選擇自動或手動產生。 為簡單起見，建議您將它設為「 **自動模式」**。 如果您選擇「自動」，則也設 **定「標籤為發佈** 」選項，否則檔案不會上線。 Sitemap和RSS檔案是XML檔案的類型，必須像發佈其他資產一樣發佈。 如果您目前尚未準備好所有資訊，或只想進行一次性產生，請使用其中一種手動模式。
4. 填入將用於檔案的中繼資料標籤。 此步驟不是可選的。 您至少必須包含三個標有星號(\*)的欄位： **著陸頁面** 、 **標題**、 **說明**。 若要使用這些工作的中繼資料，請將右側的「中繼資料」面板中的欄位拖放至表單上的對應欄位。 Dynamic Media Classic會自動將每個視訊的實際資料填入預留位置欄位。 您不必使用中繼資料欄位。 您可以改為在這裡輸入一些靜態文字，但每個視訊都會顯示相同的文字。
5. 在三個必填欄位中輸入資訊後，Dynamic Media Classic會啟用「儲存」和「儲 **存****與產生」按鈕** 。 按一下以儲存您的設定。 如果 **您處於自動模式** ，且想要讓Dynamic Media Classic稍後產生檔案，請使用「儲存」。 使用 **「儲存並產生** 」可立即建立檔案。

### 測試和發佈您的視訊網站地圖、mRSS饋送或兩個檔案

## MP4視訊的行動實作

Dynamic Media Classic不包含行動裝置的檢視器預設集，因為檢視器不需要在支援的行動裝置上播放視訊。 只要您編碼為H.264 MP4格式— 在您的桌上型電腦上轉換上傳或預先編碼， 支援的平板電腦和智慧型手機將可播放您的影片，而不需要檢視器。 這在Android和iOS（iPhone和iPad）裝置上受支援。

不需要檢視器的原因，在於這兩種平台都支援原生H.264。 您可以將視訊內嵌在HTML5網頁中，或將視訊內嵌在應用程式本身，而Android和iOS作業系統將提供控制器來播放視訊。

因此，Dynamic Media Classic不會為您提供行動裝置檢視器的URL，而是直接為視訊提供URL。 在MP4視訊的「預覽」視窗中，將會有案頭和行動裝置的連結。 行動URL會指向已發佈的視訊。

有關已發佈視訊的一個重要事項是，URL會列出視訊的完整路徑，而不只是資產ID。 處理影像時，無論檔案夾結構為何，您都會依其資產ID呼叫影像。 不過，對於視訊，您也必須指定檔案夾結構。 在上述URL中，視訊儲存在路徑中：
